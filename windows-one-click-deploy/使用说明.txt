基础命令

最简单方式：只下载一个文件并双击：

[点击下载：步骤1-一键安装脚本](https://github.com/loqwe/heyun-zjmf-worker-monitor/raw/main/windows-one-click-deploy/步骤1-一键安装脚本.bat)
这是 main 分支 raw 直链，点击后会直接下载 步骤1-一键安装脚本.bat（中文名），双击即可运行。
如果你是在 PowerShell 当前目录里手动运行，请先 `cd` 到文件所在目录，再输入 `.\步骤1-一键安装脚本.bat`；不要直接粘贴完整路径。

它会先检查 PowerShell、Node.js、npx；缺少 Node.js/npx 时会尝试通过 winget 自动安装 Node.js LTS，找不到任何 PowerShell 时会尝试通过 winget 安装 PowerShell 7，然后自动下载 步骤2-一键部署.bat、部署脚本和配置模板，并在同目录自动创建：

one-click.config.jsonc

如果你已经下载了完整目录，也可以直接双击：

步骤2-一键部署.bat

脚本会要求你输入：

Cloudflare API Token
Cloudflare Account ID
GitHub 仓库地址
ZJMF_ADMIN_TOKEN 网站密码（管理后台密码）

这些信息的获取方式：

1. Cloudflare API Token
打开 https://dash.cloudflare.com/profile/api-tokens
点击 创建令牌 -> API 令牌模板 -> 编辑 Cloudflare Workers -> 使用模板。
下一步点击 增加更多帐户，添加 D1 / 编辑。
账户资源选择 包括所有账户；区域资源选择 包括所有区域。
最后滑到最下面，点击 继续以显示摘要，再点 创建令牌。
Token 只显示一次，请复制保存，部署脚本会要求粘贴。

2. Cloudflare 账户 ID（不是 IP）
可以复制脚本检测显示的账户 ID；也可以在 Cloudflare 账户主页右侧三个点里点击 复制账户 ID。

3. GitHub 仓库地址
复制你 Fork 后仓库的地址，例如：
https://github.com/你的用户名/heyun-zjmf-worker-monitor

4. 魔方财务 API
打开 https://www.heyunidc.cn/apimanage 获取登录邮箱或手机号、API 密钥；产品 ID 可在魔方财务产品详情或部署后在管理后台填写。

可能需要：更新方式
后续更新首推再次双击 步骤1-一键安装脚本.bat。
如果想在管理后台点 系统更新 -> 确定更新，再按下面的更新方式准备 GitHub 更新令牌和 GitHub Actions Secrets。

更新方式：

1. 重新运行安装脚本
首推这个方式。双击 步骤1-一键安装脚本.bat，按提示重新部署即可。脚本会刷新源码并复用同名 D1 数据库，已有配置会保留。

2. 管理后台自动更新
进入 管理后台 -> 系统更新 -> 检查更新 / 确定更新。
这个方式需要 GitHub 更新令牌；不填写也能部署，但只能检查更新，不能在网页里点“确定更新”。
获取方式：
打开 https://github.com/settings/personal-access-tokens/new
Token name 填 zjmf-monitor-update。
Resource owner 选择你的 GitHub 账号。
Repository access 选择 Only select repositories，并选择你 Fork 后的 heyun-zjmf-worker-monitor 仓库。
Repository permissions 设置 Actions: Read and write、Contents: Read-only。
点击 Generate token 后复制 github_pat_ 开头的令牌。
注意：GitHub 更新令牌是 github_pat_ 开头，不是 Cloudflare 的 cfut_ Token。
网页自动更新依赖 GitHub Actions；你的 Fork 仓库还必须配置 CLOUDFLARE_API_TOKEN、ZJMF_ADMIN_TOKEN、CLOUDFLARE_ACCOUNT_ID 等 Secrets。

3. GitHub 同步 Fork 后自动部署
支持。到你的 Fork 仓库页面点击 Sync fork -> Update branch，或自己同步上游后 push。
只要 Fork 仓库配置好了 Actions Secrets，push 会自动触发部署。
如果你只用本地脚本部署、没有配置 GitHub Secrets，这种方式可能因为 Missing CLOUDFLARE_API_TOKEN 失败；请改用方式 1。

如果 Token 能访问多个 Cloudflare 账户，脚本只会提示检测结果，不会自动替你选择账号；请手动输入要部署的 Account ID。

ZJMF_ADMIN_TOKEN 网站密码会要求输入两次确认。两次都直接回车时，默认密码为 admin。部署后可以在管理面板的设置页修改密码。

第一次运行会自动生成本地配置文件：

one-click.config.jsonc

脚本会把确认后的管理后台密码写入配置文件，并作为 Worker Secret 注入 Cloudflare。

如果你想部署后自动添加监控项，再编辑：

zjmfApiAccount
zjmfApiPassword
serverId

如果要用 HTTP(S) + TCP + API 三步检测，建议再填写：

httpUrl
tcpHost
tcpPort

只检查环境：

pwsh -File .\deploy-one-click.ps1 -PreflightOnly

确认配置：

pwsh -File .\deploy-one-click.ps1 -PrepareOnly

正式部署：

pwsh -File .\deploy-one-click.ps1

其他命令：

强制重新下载源码：
pwsh -File .\deploy-one-click.ps1 -RefreshSource

使用本地源码：
pwsh -File .\deploy-one-click.ps1 -SourceRoot D:\你的路径\heyun-zjmf-worker-monitor

只部署，不初始化监控配置：
pwsh -File .\deploy-one-click.ps1 -SkipSeed

如果 Wrangler 新版本在 Windows 上崩溃，可指定旧版本：
pwsh -File .\deploy-one-click.ps1 -WranglerVersion 4.90.1

不想浏览器登录 Cloudflare 时，可先设置环境变量：

$env:CLOUDFLARE_API_TOKEN = "你的 Cloudflare API Token"
$env:CLOUDFLARE_ACCOUNT_ID = "你的 Cloudflare Account ID"

注意：
one-click.config.jsonc 会保存管理密码和魔方财务 API 密钥，已经加入 .gitignore，不要上传到 GitHub。
